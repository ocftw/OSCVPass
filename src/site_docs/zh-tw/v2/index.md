# OSCVPass 2.0 開源貢獻者的長期支持計畫

!!! info "參與討論"

    目前專案正在蒐集新的想法與概念，可以[到這裡](https://github.com/ocftw/OSCVPass/discussions/41){target="_blank"}加入討論。

OSCVPass 是一個從台灣出發、已累積超過十年經驗的開源貢獻支持計畫。過去，OSCVPass 以「**研討會門票回饋**」作為主要誘因，陪伴許多貢獻者踏出開源的第一步；然而，隨著環境改變，我們也觀察到這樣的模式，已不足以支撐貢獻者的長期投入與社群的持續成長。

因此，OSCVPass 2.0 誕生了。

## 從門票回饋，轉向「開源貢獻者資格認證」

在 2.0 版本中，OSCVPass 不再只是研討會的 VIP 快速通行證，而是定位為一個「**開源貢獻者資格認證**」。**透過審視近一年的實際貢獻行為，給予貢獻者一整年的開源貢獻者身份，並與研討會、社群、企業及合作夥伴共同提供多元的回饋方式**。

我們希望讓「**持續投入開源**」本身成為一件被看見、被肯定、也能被支持的事情。

## 擴大貢獻的定義：個人 × 社群 × 專案

OSCVPass 2.0 最大的改變，是將貢獻形式從單一的「個人程式碼貢獻」，擴展為三個核心面向：

- 個人：延續 1.0 的精神，鼓勵首次參與開源，也特別重視非程式碼貢獻，如文件、設計、翻譯、推廣與組織工作。
- 社群：肯定社群經營、志工投入與活動組織的價值，讓長期支撐社群運作的付出能被正式認可。
- 專案：支持開源專案的建立與成長，包含授權選擇、文件撰寫、貢獻指南與國際化準備，協助專案走向可長期發展的狀態。

透過這三種形式，我們希望讓開源貢獻不再只是短期行為，而是一條可以被陪伴的成長路徑。

## 建立正向循環，讓貢獻回到社群

OSCVPass 2.0 期待建立一個可持續的循環，而非一次性的交換。貢獻不只回饋到個人，也能回流到社群與專案，再進一步連結研討會與合作夥伴，形成彼此支持的生態系。

為此，計畫中將逐步加入：

- 貢獻者與專案的訪談專欄，累積開源故事與經驗分享。
- 與社群、研討會、店家及企業的多元合作方式。
- 穩定運作的志工團隊，支撐專案營運與對外連結。

## 2026：以「無預算」為前提，走向可持續未來

2026 年，OSCVPass 將在既有的研討會合作基礎上，進一步提升基金會在社群與人才面向的能見度，並嘗試建立能長期運作的志工團隊。在有限資源下，透過每週穩定投入與社群協力，逐步推進計畫執行。

同時，我們也將探索未來能轉化為實際資金收入或其他支持基金會的可能性，讓 OSCVPass 不只是一次改版，而是一個能持續陪伴台灣開源社群前進的計畫。

### 版本對照表

| 面向             | OSCVPass 1.0                 | OSCVPass 2.0                                           |
| ---------------- | ---------------------------- | ------------------------------------------------------ |
| **計畫定位**     | 回饋型方案（門票、快速通行） | 身份型認證（一年期開源貢獻者資格）                     |
| **主要誘因**     | 研討會門票回饋               | 多元回饋（研討會、社群、企業、合作夥伴）               |
| **貢獻時間觀點** | 偏向單次或短期               | 檢視「近一年」的實際持續貢獻                           |
| **貢獻類型**     | 以個人、程式碼貢獻為主       | 個人 × 社群 × 專案，且高度重視非程式碼貢獻             |
| **非程式碼貢獻** | 有，但不是核心               | 成為明確且被強調的重點（文件、設計、翻譯、推廣、組織） |
| **社群角色**     | 使用者／受益者               | 生態系的一部分，貢獻可回流社群與專案                   |
| **長期發展性**   | 較難支撐長期投入             | 目標建立可持續的正向循環                               |

## OSCVPass 2026 各季預定目標整理

### 2026/Q1：基礎調整與準備階段

重點在於制度調整、社群回饋蒐集與專案運作基礎建立。

- 蒐集過往申請者的意見，挑選可行方案嘗試執行，增加社群參與感
- 改寫申請說明，明確區分「個人、社群、專案」三種申請類型，並加強非程式碼貢獻的引導
- 建立潛在合作對象名單（社群、研討會、店家），規劃更多元的合作方式
- 招募至少 5 人的志工團隊，協助專案的基本營運與執行
- 持續調整 OSCVPass 專案頁面內容，補強申請說明與文件型態的資訊呈現
- 規劃並準備未來的訪談專欄方向

### 2026/Q2：新方案推廣與合作啟動

重點放在對外溝通、新制度曝光與內容產出。

- 宣傳新的申請方式與 2.0 調整方向，提供可供社群、申請者與合作店家使用的宣傳素材
- 主動招募合作對象（社群、店家），推廣與 OSCVPass 的合作模式
- 整理過往已申請的貢獻者與專案，規劃訪談主題
- 目標開始產出訪談文章，暫定一個月一篇貢獻者或專案訪談內容

### 2026/Q3：執行回饋與企業合作規劃

重點為中期檢視與企業合作模式的具體化。

- 彙整上半年專案執行的回饋，進行制度與流程的調整
- 規劃可提供給企業的合作計畫，包括：
  - OSCVPass 在企業端的角色定位（找專案、找貢獻者等）
  - 企業可編列預算的合作項目
  - 企業與 OSCVPass 合作的實際建議方案

### 2026/Q4：成果展現與年度回顧

重點在於成果累積、社群曝光與下一年度規劃。

- 舉辦年度開源專案票選活動，參考 Apple Store 分類概念，規劃多項獎項，例如：
  - 最佳協作專案
  - 最佳文章撰寫
  - 年度最佳開源專案（團隊）
  - 值得關注的開源新專案
  - 最佳個人貢獻等
- 進行年度專案回顧，蒐集社群與合作夥伴意見
- 規劃下一年度 OSCVPass 的執行方向與調整重點

### 全年執行前提與核心方向

- 持續維持與研討會的合作關係
- 提升基金會在社群與人才面向的能見度
- 建立可長期維運 OSCVPass 的志工團隊
- 在「無預算」的條件下，專案由內部成員穩定投入每週約 1～2 日作為執行基礎，並結合志工夥伴的專長與參與，一起完成專案各項工作
- 在 2026 年執行過程中，逐步探索可轉化為實際資金收入或其他贊助基金會的可能性
