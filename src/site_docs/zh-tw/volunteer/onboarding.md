# :octicons-checklist-16: 志工 Onboarding 流程

感謝加入 OSCVPass 專案志工，我們未來工作流程會以 Github 平台為主，請依照以下步驟確認是否完成相關的帳號建立與文件閱讀。

## 第一步：建立 GitHub 帳號

- 前往 [GitHub](https://github.com){target="_blank"} 註冊帳號（如已有帳號可略過）
- 完成帳號設定，建議上傳個人頭像與簡短自我介紹
- 記下你的 GitHub 使用者名稱

## 第二步：提供帳號資訊

- 將你的 GitHub 使用者名稱提供給專案聯絡人
- 等待專案團隊將你加入 [OSCVPass](https://github.com/ocftw/OSCVPass){target="_blank"} 的 GitHub Repository
- 確認收到 GitHub 的邀請通知並接受邀請

## 第三步：熟悉專案環境

- 閱讀專案的介紹文件，了解 OSCVPass 的目標與運作方式
    - [x] [OSCVPass 2.0](../v2/index.md)
    - [x] [社群合作夥伴](../partner/index.md)
    - [x] [非程式相關的開源授權](../non-software-license.md)
- 瀏覽專案 [Discussions](https://github.com/ocftw/OSCVPass/discussions){target="_blank"}，認識其他志工夥伴
- 在 [Discussions](https://github.com/ocftw/OSCVPass/discussions){target="_blank"} 的自我介紹區發文，讓大家認識你

## 第四步：了解工作內容與工具

- 查看 GitHub [Projects](https://github.com/orgs/ocftw/projects/3){target="_blank"} 看板，了解目前正在進行的工作
- 閱讀`志工手冊`，了解協作流程與使用工具
- 熟悉 [GitHub Issues](https://github.com/ocftw/OSCVPass/issues){target="_blank"}、[Pull Requests](https://github.com/ocftw/OSCVPass/pulls){target="_blank"} 等基本操作

## 第五步：選擇第一個任務

- 瀏覽標記為 `good first issue` 的任務
- 選擇符合你時間與專業的工作項目
- 在 [Issue](https://github.com/ocftw/OSCVPass/issues){target="_blank"} 中留言表示認領任務

## 第六步：開始貢獻

- 根據任務說明進行工作
- 遇到問題時在 Discussions 或 Issue 中提問
- 完成後透過 Pull Request 提交成果

## 第七步：持續參與

- 定期查看專案更新與新任務
- 參加定期會議或非同步討論
- 分享你的想法與建議，協助專案成長

!!! note "提醒"

    - 協作模式為遠端、非全職、非即時待命
    - 建議每週投入約 1~3 小時
    - 有任何疑問都歡迎隨時提出！
