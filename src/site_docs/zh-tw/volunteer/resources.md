## 範本與參考

### 文件範本

#### Issue 範本

認領任務時的留言範本：

```markdown
我想認領這個任務！

預計完成時間：[例如：本週末、下週三]

相關問題：
- [若有疑問可以在這裡提出]

謝謝！
```

#### Pull Request 範本

提交 PR 時的描述範本：

```markdown
## 變更說明

[簡短描述你做了什麼]

## 相關 Issue

Fixes #[Issue 編號]

## 變更內容

- [ ] 新增 XX 功能
- [ ] 修正 YY 問題
- [ ] 更新 ZZ 文件

## 截圖或範例

[如適用，附上截圖或範例]

## Checklist

- [ ] 我已經測試過這些變更
- [ ] 我已經檢查錯字與格式
- [ ] 我已經更新相關文件（如需要）
```

### 腳本與工具

| 腳本 | 位置 | 用途 | 使用說明 |
|------|------|------|---------|
| `update_approved_from_vcs.py` | `scripts/` | 更新已核准專案詳細資料 | 參考 [README.md](../../README.md) |

**常用指令**（需在 `src/site_docs/` 目錄執行）：

```bash
# 本地端預覽網站
uv run mkdocs serve

# 建置網站
uv run mkdocs build

# 更新專案詳細資料（預覽）
uv run python scripts/update_approved_from_vcs.py

# 更新專案詳細資料（寫入）
uv run python scripts/update_approved_from_vcs.py --write
```
